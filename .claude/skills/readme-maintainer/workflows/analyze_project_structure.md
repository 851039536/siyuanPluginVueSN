# 项目结构分析工作流

这个工作流帮助您分析当前项目的整体结构，为README文档提供基础信息。

## 分析步骤

### 1. 项目基本信息收集 ⭐⭐⭐⭐⭐
- [ ] 确定项目名称和版本
- [ ] 识别项目类型（Web应用、库、工具等）
- [ ] 了解项目的主要目的和解决的问题
- [ ] 确定目标用户群体

### 2. 目录结构分析 ⭐⭐⭐⭐
- [ ] 扫描项目根目录的主要文件和文件夹
- [ ] 识别常见的项目结构模式
- [ ] 分析配置文件（package.json, requirements.txt等）
- [ ] 检查文档目录是否存在

### 3. 技术栈识别 ⭐⭐⭐⭐
- [ ] 识别主要编程语言
- [ ] 检查框架和库的依赖
- [ ] 分析构建工具和开发工具
- [ ] 识别数据库和存储方案

### 4. 代码组织分析 ⭐⭐⭐
- [ ] 分析源代码目录结构
- [ ] 检查模块划分是否清晰
- [ ] 查看测试文件组织
- [ ] 分析静态资源管理

### 5. 开发流程检查 ⭐⭐⭐
- [ ] 检查版本控制配置
- [ ] 分析CI/CD配置
- [ ] 查看代码质量工具配置
- [ ] 检查部署配置

## 输出建议

基于分析结果，建议在README中包含：

1. **项目概述** - 简要说明项目目的和功能
2. **技术栈** - 列出主要使用的技术和工具
3. **项目结构** - 提供目录结构说明
4. **开发环境** - 说明如何设置开发环境
5. **构建和部署** - 说明如何构建和部署项目

## 下一步操作

完成分析后，您可以：
1. 使用这些信息更新现有README
2. 运行 `generate_readme_template.md` 生成新模板
3. 针对特定问题进行调整
