## 产品概述

将 EditProjectDialog.vue 编辑弹窗中的"仓库链接"区块从固定 4 输入框平铺改为列表式管理，交互方式与同弹窗内"Git 远程仓库"区块保持一致。

## 核心功能

- 列表展示已配置的仓库链接（平台图标 + 平台名 + URL），仅显示有值的条目
- 每条链接支持单独编辑（内联 Input + 保存/取消按钮）和删除
- 底部添加行：Select 选择尚未配置的平台 + URL 输入框 + 添加按钮
- 每次新增/编辑/删除立即持久化保存（调 `updateProjectMeta`），操作即时生效
- 每个平台最多一个链接，保持现有 `githubUrl/giteeUrl/giteaUrl/cnbUrl` 四字段数据模型不变

## 技术栈

- 框架：Vue 3 + TypeScript（现有项目，无新增依赖）
- 样式：SCSS（复用现有 `.gp-remote-*` 类，Codex UI 风格）
- 数据持久化：`GitPushManager.updateProjectMeta()` + `TypedStorage`

## 实现方案

### 策略

将"仓库链接"区块的模板从 `v-for PLATFORM_META` 平铺 4 个 Input 改为列表式 UI，完全镜像同弹窗内"Git 远程仓库"区块的结构（已配置列表 + 内联编辑 + 底部添加行）。每次增删改操作立即调用 `updateProjectMeta` 持久化对应平台字段，并 `emit("saved")` 通知父组件刷新卡片。

### 数据流

```
用户操作（添加/编辑/删除）
  → 更新 urlInputs reactive 对应字段
  → 调用 saveRepoLink(urlProp)
  → updateProjectMeta(projectId, { [urlProp]: value | undefined })
  → invalidateProjectCache()（manager 内部）
  → emit("saved") → 父组件刷新项目列表（卡片平台图标/链接同步更新）
```

### 关键技术决策

1. **复用 PLATFORM_META 常量**：`urlProp` 字段（"githubUrl" 等）直接作为 `urlInputs` 的 key 和 `updateProjectMeta` 的 patch key，无需新增类型
2. **复用 `.gp-remote-*` 样式类**：两个区块视觉一致，避免新增样式。行内增加平台 `<Icon>` 元素即可
3. **computed 派生列表**：`repoLinkList` 从 `urlInputs` 过滤非空条目；`availablePlatforms` 过滤未配置平台供添加行 Select 使用
4. **立即保存 + emit("saved")**：与 Git 远程仓库操作一致（每次操作即时生效），且仓库链接变更影响卡片展示需刷新父组件
5. **从 save() 移除 urlInputs**：因每次操作已独立持久化，主保存按钮不再重复保存 url 字段

### 类型安全处理

`updateProjectMeta` 的 patch 类型为 `Partial<Pick<GitProject, "githubUrl" | "giteeUrl" | "giteaUrl" | "cnbUrl" | ...>>`。使用计算属性名 `{ [urlProp]: value }` 时 TypeScript 无法从 `string` 推断为联合类型，需将参数标注为 `PLATFORM_META[number]["urlProp"]` 联合类型或使用类型断言。

## 实现注意事项

- **emit("saved") 不关闭弹窗**：与主 `save()` 一致，"saved" 仅通知父组件刷新数据，弹窗保持打开，用户可连续操作多个链接
- **删除即清空**：删除链接 = 将对应字段设为 `undefined`（`updateProjectMeta` 中 `Object.assign` 会写入 undefined，持久化时 JSON.stringify 自动忽略 undefined 字段）
- **添加行禁用态**：`!newLinkPlatform || !newLinkUrl.trim()` 时禁用添加按钮，与远程仓库添加行一致
- **全部平台已配置时**：添加行 Select 无可选项，应隐藏添加行或禁用
- **编辑取消不影响数据**：内联编辑使用独立 `editLinkUrl` ref，取消时不写回 `urlInputs`，与远程仓库编辑模式一致
- **if 语句必须带花括号**（项目代码风格规则）
- **文件头注释**：本文件已有注释，无需修改

## 架构设计

无新增架构模式。改动局限于单个 Vue 组件内部（模板 + 脚本），复用现有 `PLATFORM_META` 常量、`updateProjectMeta` 方法、`.gp-remote-*` 样式类。数据模型（GitProject 四字段）不变，无跨模块影响。
