# AI信息生成调试指南

## 问题1：DeepSeek没有实时显示输出

### 调试步骤

1. **打开浏览器控制台**（F12）

2. **配置DeepSeek API**
   - 打开超级面板
   - 点击设置图标
   - 选择API供应商：DeepSeek
   - 选择模型：DeepSeek Chat
   - 输入API密钥
   - 保存配置

3. **测试生成**
   - 打开AI信息生成面板
   - 输入一个简单的问题，如："你好"
   - 点击生成
   - 观察控制台输出

### 预期的控制台日志

```
开始生成内容，配置: {provider: "deepseek", model: "deepseek-chat", hasOnChunk: true}
DeepSeek流式API请求开始
DeepSeek响应状态: 200
DeepSeek流式输出块1: 你好
DeepSeek流式输出块2: ！
收到流式数据块: 你好
收到流式数据块: ！
DeepSeek流式读取完成，总块数: 2
DeepSeek流式输出完成，总长度: 3
生成完成，响应长度: 3
```

### 可能的问题和解决方案

#### 问题A：没有看到"DeepSeek流式API请求开始"
**原因**：可能没有正确调用流式API
**解决**：检查是否正确配置了API供应商为DeepSeek

#### 问题B：看到"DeepSeek响应状态: 200"但没有流式输出
**原因**：DeepSeek API可能不支持流式输出，或者响应格式不同
**解决**：
1. 检查API密钥是否正确
2. 查看是否有"解析DeepSeek SSE数据失败"的警告
3. 如果有警告，查看具体的line内容，可能需要调整解析逻辑

#### 问题C：内容生成完成但界面不显示
**原因**：Vue响应式更新问题
**解决**：检查是否有"收到流式数据块"的日志，如果有但界面不更新，可能是Vue组件问题

## 问题2：提示词和文档引用没有关联

### 调试步骤

1. **测试提示词功能**
   
   a. 打开AI信息生成面板
   
   b. 点击"对话设置"
   
   c. 修改系统提示词为：
   ```
   你是一个专业的技术文档编写助手。请用简洁、专业的语言回答问题，并使用Markdown格式。
   ```
   
   d. 点击"保存当前配置"，输入名称："技术助手"
   
   e. 关闭对话设置
   
   f. 点击"选择提示词"，选择"技术助手"
   
   g. 观察控制台输出：
   ```
   ✓ 已加载配置: 技术助手
   ```
   
   h. 输入问题："什么是Markdown？"
   
   i. 点击生成
   
   j. 观察控制台输出：
   ```
   系统提示词: 你是一个专业的技术文档编写助手...
   用户输入: 什么是Markdown？
   构建提示词，无上下文，长度: XX
   ```

2. **测试文档引用功能**
   
   a. 在思源笔记中打开一个文档
   
   b. 打开AI信息生成面板
   
   c. 点击"@当前文档"按钮
   
   d. 观察控制台输出：
   ```
   文档引用成功: {title: "文档标题", contentLength: 1234, contentPreview: "..."}
   ```
   
   e. 验证界面显示：应该看到"📄 文档标题"

3. **测试提示词+文档引用**
   
   a. 确保已选择提示词（如"技术助手"）
   
   b. 确保已引用文档
   
   c. 输入问题："总结这个文档的主要内容"
   
   d. 点击生成
   
   e. 观察控制台输出：
   ```
   系统提示词: 你是一个专业的技术文档编写助手...
   已添加文档上下文，标题: 文档标题, 内容长度: 1234
   用户输入: 总结这个文档的主要内容
   构建完整提示词，包含上下文，长度: XXXX
   ```

### 预期行为

当同时使用提示词和文档引用时：

1. **系统提示词**（System Prompt）：定义AI的角色和行为方式
   - 例如："你是一个专业的技术文档编写助手"

2. **用户消息**（User Message）：包含文档内容和用户问题
   ```
   上下文信息:

   **当前文档信息**：
   文档标题：XXX
   文档内容：
   [文档的完整内容]

   请基于以上文档内容回答用户问题。

   用户问题:
   总结这个文档的主要内容
   ```

3. **AI响应**：应该基于系统提示词的角色，结合文档内容来回答问题

### 可能的问题和解决方案

#### 问题A：提示词没有生效
**症状**：AI的回答风格与提示词不符
**调试**：
1. 检查控制台是否有"系统提示词:"日志
2. 确认提示词内容是否正确
3. 尝试使用更明确的提示词，如："你必须用中文回答，并且每个回答都要以'作为技术助手，'开头"

#### 问题B：文档内容没有被使用
**症状**：AI的回答与文档内容无关
**调试**：
1. 检查控制台是否有"已添加文档上下文"日志
2. 检查"构建完整提示词，包含上下文"日志
3. 确认文档内容长度不为0
4. 尝试更明确的问题，如："文档中提到了哪些关键点？"

#### 问题C：文档内容过长导致截断
**症状**：只有部分文档内容被使用
**解决**：
1. 检查maxTokens设置（在对话设置中）
2. 增加maxTokens值（如4000）
3. 或者在问题中明确指出要关注的部分

## 通用调试技巧

### 1. 清除缓存
如果遇到奇怪的问题，尝试：
- 清除浏览器缓存
- 重新加载插件
- 重启思源笔记

### 2. 检查API配置
确保：
- API密钥正确
- 选择了正确的供应商和模型
- 网络连接正常

### 3. 使用简单测试
先用最简单的输入测试：
- 不使用提示词
- 不引用文档
- 只输入"你好"
- 确认基本功能正常后再测试复杂场景

### 4. 对比不同供应商
如果DeepSeek有问题，尝试：
- 切换到通义千问测试
- 对比控制台日志差异
- 确定是API特定问题还是通用问题

## 报告问题时请提供

1. **控制台完整日志**（从点击生成到完成的所有日志）
2. **API配置信息**（供应商、模型，不要包含API密钥）
3. **输入内容**（提示词、用户问题、是否引用文档）
4. **预期行为** vs **实际行为**
5. **浏览器和思源笔记版本**
