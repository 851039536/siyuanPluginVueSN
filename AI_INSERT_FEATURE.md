# AI 内容生成器 - 插入到当前文档功能

## 功能说明

在普通模式下，AI 内容生成器现在支持将生成的内容直接插入到当前文档中。

## 新增功能

### 1. 插入按钮
- **位置**：生成内容后，在结果区域的操作按钮中
- **显示条件**：仅在普通模式（非编辑模式）下显示
- **样式**：绿色高亮按钮，易于识别

### 2. 插入流程
1. 生成 AI 内容后，点击"插入"按钮
2. 系统弹出确认对话框：**"确定要将生成的内容插入到当前文档吗？这将覆盖文档的现有内容。"**
3. 确认后，内容将覆盖当前光标所在文档的内容
4. 显示成功提示：**"✓ 已插入到文档: [文档名称]"**

### 3. 撤回功能
- 插入后，会显示"撤回"按钮
- 点击撤回可以恢复文档的原始内容
- 撤回功能与编辑模式共用同一个历史记录机制

## 使用场景

### 场景 1：基于文档标题生成完整内容
1. 创建一个只有标题的新文档（如 "Tesseract"）
2. 打开 AI 内容生成器
3. 点击"@当前文档"引用该文档
4. 设置系统提示词（如技术文档撰写规范）
5. 点击"生成"（无需输入问题，会自动使用默认提示）
6. 生成完成后，点击"插入"按钮
7. 确认后，文档内容被替换为 AI 生成的完整技术文档

### 场景 2：生成内容后插入到其他文档
1. 在 AI 内容生成器中生成内容
2. 切换到目标文档，将光标放在文档中
3. 返回 AI 内容生成器
4. 点击"插入"按钮
5. 内容将插入到当前光标所在的文档

## 技术实现

### 核心函数
- `insertToCurrentDocument()` - 主插入函数
- `insertContentToDocument(docId)` - 执行插入操作
- `getCurrentBlockId()` - 获取当前光标位置
- `getDocIdByBlockId()` - 通过块 ID 获取文档 ID

### 安全机制
1. **确认对话框** - 防止误操作
2. **历史记录** - 保存原始内容用于撤回
3. **错误处理** - 完善的错误提示和异常处理

### 状态管理
- `isInserting` - 插入进行中状态
- `lastEditHistory` - 最后一次编辑/插入的历史记录

## 国际化支持

### 中文
- `insertToDocument`: "插入到当前文档"
- `confirmInsertToDocument`: "确定要将生成的内容插入到当前文档吗？这将覆盖文档的现有内容。"
- `undoInsert`: "撤回插入"

### 英文
- `insertToDocument`: "Insert to Current Document"
- `confirmInsertToDocument`: "Are you sure you want to insert the generated content into the current document? This will overwrite the existing content."
- `undoInsert`: "Undo Insert"

## 与编辑模式的区别

| 特性 | 普通模式 + 插入 | 编辑模式 |
|------|----------------|----------|
| 目标文档选择 | 自动（当前光标所在文档） | 手动选择 |
| 差异对比 | 不支持 | 支持 |
| 内容编辑 | 不支持 | 支持 |
| 应用方式 | 一键插入 | 查看差异后应用 |
| 使用场景 | 快速插入生成内容 | 精细编辑现有文档 |

## 注意事项

1. **数据安全**：插入操作会覆盖文档内容，请确保重要内容已备份
2. **撤回限制**：只能撤回最后一次插入操作
3. **光标位置**：确保光标在目标文档中，否则可能无法获取文档信息
4. **权限要求**：需要文档编辑权限
