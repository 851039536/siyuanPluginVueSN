## 🔧 已实施的改进

### 问题 1: 快捷键面板图标不直观

**原始状态**
```typescript
// src/features/shortcut/index.ts (第 270 行)
icon: 'iconHelp'  // 问号图标 ❌
```

**问题分析**
- ❌ `iconHelp` (问号) 会让用户误认为是帮助或常见问题
- ❌ 不能直观表达"快捷键"的含义
- ❌ 容易引起功能理解混淆

**改进方案**
```typescript
// src/features/shortcut/index.ts (第 270 行)
icon: 'iconKeymap'  // 键盘图标 ✅
```

**优化后效果**
- ✅ 键盘图标直观地表达快捷键功能
- ✅ 用户一目了然地知道这是快捷键面板
- ✅ 与思源笔记的设计风格保持一致

## 📐 图标选择指南

### TopBar（顶部栏）图标建议

| 功能类型 | 推荐图标 | 说明 |
|---------|--------|-----|
| 图片编辑 | `iconImage` | 表示与图片相关的操作 |
| 文档操作 | `iconFile` | 表示文档或文件操作 |
| 工具设置 | `iconSettings` | 表示配置或设置功能 |
| 快捷键 | `iconKeymap` | 表示键盘快捷键 |
| 搜索替换 | `iconSearch` | 表示搜索功能 |

### Dock（右侧栏）图标建议

| 功能类型 | 推荐图标 | 说明 |
|---------|--------|-----|
| 快捷键面板 | `iconKeymap` | 键盘图标，清晰直观 |
| 文档锁定 | `iconLock` | 锁形图标，表示安全保护 |

### ⚠️ 应避免使用的图标

| 图标 | 原因 | 示例 |
|-----|------|------|
| `iconHelp` | 易误导用户，表示帮助而非功能 | ❌ 快捷键面板不应用问号 |
| `iconQuestion` | 同上，增加用户困惑 | ❌ 不要用于表示具体功能 |
| `iconInfo` | 表示信息提示，不应用于功能 | ❌ 除非是信息面板 |
| `iconWarning` | 表示警告，通常用于错误提示 | ❌ 不应用于正常功能 |

## 🎨 设计原则

### 1. 准确性原则
- 图标应该准确反映功能的核心含义
- 避免使用通用的抽象图标（如？、！等）
- 优先使用具有明确含义的图标

### 2. 一致性原则
- 与思源笔记官方图标库的设计风格保持一致
- 在整个插件中使用统一的视觉语言
- 相同功能类型使用统一的图标风格

### 3. 直观性原则
- 用户应该能快速理解图标的含义
- 不需要额外学习或记忆
- 遵循用户的常见认知（如锁表示安全、齿轮表示设置等）

### 4. 可辨识性原则
- 在侧边栏中的多个图标应有足够的视觉区分
- 不应与思源笔记内置功能的图标过于相似
- 避免图标过于复杂难以辨认
